# EEZ Messaging Guide v1 (Draft)

## Category Framing
- **The Ethereum Economic Zone (EEZ):** The shared security, liquidity, and governance layer for the next generation of Ethereum rollups.
- **Synchronous Composability:** The ability for protocols on different rollups to interact within the same transaction slot, removing the need for bridges and fragmented liquidity.
- **One Ethereum:** Restoring Ethereum's original vision of a unified ecosystem, but with the scalability of the L2 roadmap.

## Approved Language
- "EEZ is governed by the EEZ Association, a Swiss non-profit."
- "Shared security via real-time ZK proving."
- "Atomic, synchronous composability."
- "Restoring Ethereum's network effects."
- "Neutral infrastructure for the rollup-centric roadmap."

## Forbidden Language
- "Gnosis product" / "Gnosis interoperability layer" / "Gnosis Chain's new feature" (EEZ is a separate, association-governed entity)
- "Bridges" (EEZ eliminates the need for them)
- "Wrapped tokens" (in the context of EEZ's native asset movement)
- "Soon" (always provide specific technical milestones or say "on track for [milestone]")

## Segment-Specific Variants
- **For Rollup Teams:** Focus on sovereignty, shared security cost reduction, and access to the EEZ liquidity pool.
- **For DeFi Protocols:** Focus on synchronous composability, capital efficiency, and removing bridge risk.
- **For Ethereum Researchers:** Focus on real-time ZK proving, slot constraints, and alignment with the rollup-centric roadmap.
